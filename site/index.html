Everything you expect from a real ORM

Typed to the core, honest about SQL, and portable across four dialects.

◇

Typed, safe queries

A Proxy expression recorder captures x.age.gt(18) into an IR — no toString() parsing, minification-safe. Illegal operators don't compile.

⟳

Change tracking & UoW

A snapshot-diffing identity map, atomic saveChanges() with FK ordering, optimistic concurrency, and ambient transactions.

⇄

Relationships & loading

Include/ThenInclude as split queries (no cartesian explosion), plus explicit and opt-in lazy loading.

△

Snapshot migrations

A differ compares your model to the last committed snapshot — never the live DB — and emits reversible, mergeable migrations.

◈

Plugins

Soft-delete, timestamps, and multitenancy ship first-party, built entirely on the same public plugin surface you'd use.

⚡

Kysely engine, hidden

Deterministic, parameterized SQL per dialect. The engine is a swappable seam — it never leaks into your types.

Write LINQ, get SQL

Compose immutable queries with a fluent, fully-typed API. Every operator maps to real SQL; anything untranslatable throws at build time, never silently.

  • Operators — eq/in/between, startsWith/contains/like/toLower/toUpper.
  • Terminals — toList, first, single, count, sum/avg/min/max, toPage.
  • Escape hatch — parameterized fromSql tagged templates.
query.ts
const page = await db.orders
  .where(x => x.total.gte(100).and(x.status.neq('cancelled')))
  .orderByDescending(x => x.createdAt)
  .select(x => ({ id: x.id, total: x.total }))
  .toPage(1, 20);

Save like a Unit of Work

Track POCO entities, mutate them, and persist in one atomic call. Ormit detects changes, orders inserts and deletes by the FK graph, writes generated keys back, and enforces concurrency tokens.

  • Identity map — find() and query results dedupe to one instance.
  • Concurrency — a stale update throws ConcurrencyError.
  • Transactions — database.transaction() is ambient (AsyncLocalStorage).
